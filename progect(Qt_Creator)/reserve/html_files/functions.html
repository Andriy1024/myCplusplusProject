Функції у C++


Об’явлення та визначення функцій

Функція – це іменована послідовність операторів, що виконує якунебудь закінчену дію. Функція може одержувати параметри і повертати єдине значення. Будь-яка функція в програмі повинна бути об’явлена і визначена. Об’явлення функції повинне бути зроблено до її виклику, щоб компілятор міг правильно обробити її параметри.

Визначення функції (прототип функції) задає її ім'я, тип значення, що повертається, і список одержуваних параметрів. Після визначення слідує тіло функції – складений оператор, що містить оператори, які визначають дію функції.

Формат визначення функції:

  тип ім'я_функції(список_параметрів)   

  {тіло функції   

  return значення}   

Тип значення, що повертається функцією може бути будь-яким, крім масиву і функції (але може бути вказівником на масив або функцію). Якщо функція не повинна повертати значення, то вона повинна мати тип void.

Список_параметрів визначає величини, що потрібно передати у функцію при її виклику. Елементи списку параметрів розділяються комами. Для кожного параметра вказується його тип і ім'я.

У визначенні, в оголошенні і при виклику однієї і тієї ж функції типи і порядок слідування параметрів повинні збігатися. Для виклику функції потрібно вказати її ім'я, а за ним у круглих дужках – імена переданих аргументів (через кому). Виклик функції може розташовуватися в будь-якому місці програми. Виключення – функція типу void не може входити до складу виражень або розташовуватися в правій частині оператора присвоювання.

Усі величини, описані у функції, а також її параметри являються локальними. Їхня область дії обмежена межами цієї функції. При виході з функції значення локальних змінних губляться. Якщо цього потрібно уникнути, при описі локальних змінних використовується модифікатор static.

Обмін даними між функціями

Обмін даними між функціями може виконуватися наступними шляхами:

1) за допомогою глобальних змінних;

2) через значення, що повертається.

3) через параметри;

Глобальні змінні

Глобальні змінні доступні усередині будь-якої функції, де немає однойменних локальних змінних. Але їх використовувати не рекомендується, тому що це утрудняє налагодження і робить функцію залежною від іншої програми (неможливість виділити функцію в окрему бібліотеку).

Значення, що повертається. Повернення з функції в точку її виклику здійснюється оператором:

   return вираження;   

У функції може бути декілька операторів return. Якщо функція описана як void, вираження не вказується (може навіть бути відсутнім і оператор return).

Вираження, зазначене в операторі return неявно перетвориться до типу значення, що повертається функцією, і передається в точку виклику. Забороняється повертати з функції вказівник на локальну змінну, тому що пам'ять, виділена циєї змінної при виході з функції звільняється.

Параметри функції

Параметри, зазначені в заголовку при описі функції являються формальними. Параметри, задані при виклику функції, називаються фактичними (аргументами).

Способи передачі параметрів у функцію:

1) за значенням;

2) за адресою (по посиланню).

При передачі за значенням функція одержує копії значень параметрів.

Можливість змінити самі параметри відсутня. При передачі за адресою функція одержує адреси аргументів. При цьому функція може змінити значення аргументів. Якщо потрібно заборонити зміну параметра усередині функції, використовується модифікатор const.

Код програми 1: Функція для обміну двох цілих чисел.

  #include <iostream.h>   

   void swap(int &x, int &y)   

   {   

    int z;   

    z=x;   

    x=y;   

    y=z;   

   }   

   void main()   

   {   

    int a=20,b=40;  

    cout<<"Значення до обміну"<<endl;   

    cout<<"a="<<a<<endl;   

    cout<<"b="<<b<<endl;   

    swap(a,b);   

    cout<<"Значення після обміну"<<endl;   

    cout<<"a="<<a<<endl;   

    cout<<"b="<<b<<endl;   

   }   

Примітка – Аргумент може передаватися або як вказівник, або як посилання. При передачі по посиланню усередині функції при звертанні до параметра неявно виконується операція розіменування. При використанні вказівника програміст повинний явно задавати операцію розіменування.

Рекурсія

Рекурсія – це здатність функції викликати саму себе.

Пряма рекурсія – коли функція викликає сама себе безпосередньо.

Непряма рекурсія – коли функція викликає себе через іншу функцію.
Для нормального завершення будь-яка рекурсивна функція повинна містити хоча б одну нерекурсивну галузь, що закінчується оператором повернення.
